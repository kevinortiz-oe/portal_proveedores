### PROVIDER 11673 SPECIFIC RULES
- **Code Identification**: 
    - The product code (SKU) is usually short (e.g., "LT-LTC/150W-W" or "Z12-LT-LTC/150W-W").
    - The description is a long text (e.g., "LAMPARA DE EMERGENCIA LED 4W...").
    - **CRITICAL**: ONLY map the short SKU to the `codigo` field. If you cannot find a short SKU, leave `codigo` as null. **NEVER** put the full description in the `codigo` field.
- **OCR Robustness**: This provider's invoices often have OCR noise. Be careful with:
    - "1" vs "I" (e.g., `Z12-` might appear as `ZI2-`).
    - "5" vs "S" (e.g., `150W` might appear as `1S0W`).
    Try to infer the correct numeric or character value based on context.
- **Column Mapping**:

    - **PRECIO UNITARIO**: Map to `valorUnitario`.
    - **IMPUESTOS**: Map to `montoImpuesto` field for each item.
    - **IMPORTE TOTAL**: Map to `importe` field for each item.
- **Totals**:
    - Extract the overall **SUBTOTAL** (sum of prices * quantity) to the `subtotal` header field.
    - Extract the overall **TOTAL IMPUESTOS** (sum of all item taxes) to the `total_impuestos` header field.
    - Extract the overall **TOTAL** to the `total` header field.
- **DANGER**: 
    - Ensure `montoImpuesto` is extracted for every line. If it is 0, enter 0.
    - The prices might have 3 decimal places (e.g., 21.380). Extract with full precision.
